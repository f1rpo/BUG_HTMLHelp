How to Install

Installation instructions for the BUG mod have been deleted from this page. Since most changes in Taurus involve the GameCore DLL, the separation between BUG as a non-DLL mod playable on MacOS and BULL as a less portable extension is discontinued - which simplifies installation.

Taurus is distributed only as a ZIP archive that needs to be extracted into the BtS Mods folder - either under Program Files or under My Games. BtS version 3.19 is required. The mod only runs on Windows or, possibly, on a Windows compatibility layer such as Wine (but no technical support can be provided for such non-native installations). Steam users will have to revert to the GameSpy "beta" version to use Taurus; a Mods folder is located in Steamapps\Common\Sid Meier's Civilization IV Beyond the Sword\Beyond the Sword.

To start the mod through a Windows shortcut, create a shortcut to Civ4BeyondSword.exe, open the shortcut's file properties, and add a space and mod=\Taurus to the Target field. Installation into CustomAssets should be possible but is not recommended as mods in CustomAssets will interfere with mods in the Mods folder that fail to set NoCustomAssets=1 in their INI file. An important benefit of installing BUG and BULL as custom assets was savegame-compatibility with BtS; Taurus is able to load and export BtS, BUG, BULL and other savegames; see the Save Compatibility help page.

Taurus will not prevent artwork under CustomAssets from being loaded and should be fully compatible with the Blue Marble mod and its Gold theme installed as custom assets. The Blue Marble terrain graphics can also be obtained as a FPK file from the archive linked at the end of this post in the K-Mod CivFanatics forum. Taurus will load any FPK files placed in Taurus\Assets. A similar pack of terrain textures, closer to the original BtS textures, can be obtained here from the Advanced Civ mod.
