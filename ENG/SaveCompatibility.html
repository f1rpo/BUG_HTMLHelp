Savegame Compatibility

BtS, Taurus, BUG, BULL and many other lightweight mods use the same savegame format (save version 302). The only thing standing in the way of savegame compatibility is a check for the mod name performed by Civ4BeyondSword.exe, a file that, generally, can't be modified. Taurus uses a hack to work around this problem and will load BtS savegames and savegames created by any mod whose name starts with either "Taurus", "BUG", "BULL" or "BAT". These name checks ignore letter case, and the list of name prefixes can be extended in Taurus\Assets\XML\GlobalDefinesAlt.xml. One can also hold down the x key at the start of the loading process to force Civ4BeyondSword.exe to accept the savegame regardless of the mod name. This will likely result in a crash to the desktop when the savegame has an incompatible format.

BUFFY (Hall of Fame) savegames cannot be loaded by Taurus because they're encrypted.

When loading a non-Taurus savegame that has the Lock Modified Assets option enabled, Taurus will disable that option and bypass the modified-assets check in Civ4BeyondSword.exe. (That check would prevent the savegame from being loaded by any mod whose files differ in any way from those in the mod that created the savegame.)

By disabling the Save Mod Name option (enabled by default) on the Saves tab of the mod options screen (Ctrl+Alt+O), Taurus can be prevented from writing its own name into savegames, thus allowing BtS (possibly with a mod installed under CustomAssets) to load Taurus savegames. Moreover, the key combination Ctrl+Shift+S can be used to bring up a dialog for exporting the current game state into a savegame that either BtS or a particular (save version 302) mod will accept. For exporting to BtS, leave the text field empty; for exporting to another mod, enter the name of the mod (i.e. the name of its root folder). Exporting to BAT is possible too. The BAT savegame format differs a bit from 302, so Taurus will write the necessary extra data when the entered mod name starts with "BAT".

A frequently requested change in any DLL mod is a higher player limit. Unfortunately, this would break savegame compatibility, so, at best, an alternative CvGameCoreDLL.dll with, say, a 32-player limit could be provided. At this time, no such a DLL is not being distributed.
